Here, I have included small to medium sized relational database projects, each typically consisting of 15–30 tables.

Each project includes:
- An introduction to the system the database is designed for.
- The business requirements or user actions the database must support.
- An Entity Relationship Diagram (ERD) showing the tables, attributes, relationships, and keys.
- SQL scripts for creating the database, including tables with appropriate constraints and metadata, indexes, views, transactions, triggers, and functions where needed.
- Seed data for testing.
- Sample SQL queries to verify that the database works as intended.

Note: Each design is reviewed to ensure the relationships are correct and the schema is normalized to Third Normal Form (3NF).
